### Phase 2 — landed

- **2A done**: `Request` in `crates/ctxd-cli/src/protocol.rs` now carries
  `PeerHello`, `PeerWelcome`, `PeerReplicate`, `PeerAck`,
  `PeerCursorRequest`, `PeerCursor`, `PeerFetchEvents`. Round-trip
  serde covered by `all_request_variants_serialize`. The
  `handle_connection` path now dispatches each variant through
  `PeerManager` when federation is attached.
- **2B done**: `ctxd peer add --url` now performs a real
  `PeerHello` ↔ `PeerWelcome` handshake. `--manual` preserves the
  v0.2 record-only path. `CTXD_FEDERATION_AUTO_ACCEPT={true|false|allowlist:<hex>...}`
  controls inbound acceptance. ADR 008 documents the decision.
- **2C done**: `CapEngine::attenuate_with_block` (third-party block,
  authority-signed) + `verify_multi(trusted_authorities)` with the
  loud-fail rule on missing trust-set entries. Spike test
  `crates/ctxd-cap/tests/third_party_chain.rs` covers the
  three-authority happy path plus four adversarial cases (wrong key,
  missing intermediate, widening, expiry).
- **2D done**: `crates/ctxd-cli/src/federation.rs::PeerManager` owns
  per-peer enrolment, signature + cap-scope verification on every
  inbound replicate, and the loop-guard rule (ADR 009). `Store::append`
  now preserves caller-supplied predecessorhash + signature so
  replicated events round-trip byte-identically.
- **2E done**: `PeerManager::catch_up_peer` reads the receiver's
  cursor via `PeerCursorRequest` and replays events past it.
  `PeerManager::backfill_parents` resolves missing parents via
  `PeerFetchEvents` with topological-sort apply. ADR 010 captures
  the design.
- **2F done**: KV view in `ctxd-store-sqlite` now enforces LWW on
  `(time, event_id)` via a conditional ON CONFLICT clause.
  Integration test `federation_concurrent_writes.rs` pins the
  invariant.
- **2G done**: `ctxd peer add|list|status|remove|grant` CLI in place
  since v0.3 Phase 1; `peer add` extended in 2B.
- **2H done**: `docs/federation.md` (concept + tutorial +
  troubleshooting), ADRs 008/009/010.

### Phase 1A — landed

- `ctxd-store-core` crate owns the `Store` trait, shared DTOs, and the conformance
  testsuite under `crates/ctxd-store-core/src/{lib.rs,testsuite.rs}`.
- `ctxd-store-sqlite` implements `Store` for `EventStore` via
  `crates/ctxd-store-sqlite/src/store_trait.rs`. The concrete `EventStore` API
  is unchanged — downstream crates (`ctxd-mcp`, `ctxd-http`, `ctxd-cli`,
  `ctxd-adapter-fs`) still import the concrete type from the `ctxd-store`
  re-export shim. Migrating those to `Arc<dyn Store>` is deferred; federation
  work in Phase 2 can depend on the trait directly where it helps.
- `crates/ctxd-store/` is a minimal back-compat shim that re-exports from
  `ctxd-store-sqlite`. No runtime code changed for downstream crates.
- Conformance test at `crates/ctxd-store-sqlite/tests/conformance.rs` runs
  every shared test against an in-memory SQLite store. Add new trait methods
  by (a) adding to `Store` in `ctxd-store-core/src/lib.rs`, (b) adding a
  conformance test in `ctxd-store-core/src/testsuite.rs`, and
  (c) wiring the impl in `ctxd-store-sqlite/src/store_trait.rs`.
