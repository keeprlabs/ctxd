  ships a real PAT-authed poller. Modules: `config`, `state`

  (reqwest + rate-limit + If-None-Match), `parse` (Link header,
  Retry-After), `events` (subject + payload + classify),
  `poller` (issues + pulls + comments + notifications), `adapter`

  subcommands). Ten wiremock-driven integration tests cover PAT
  headers, issue/PR/comment polling, PR lifecycle, pagination,
  ETag 304 short-circuit, primary + secondary rate limits,
  notifications, and idempotency on restart. `docs/adapters/github.md`
  documents PAT setup, subject scheme, payload shape, rate-limit
  behavior, state DB layout, and troubleshooting.

- **2A done**: `Request` in `crates/ctxd-cli/src/protocol.rs` now carries

  `PeerCursorRequest`, `PeerCursor`, `PeerFetchEvents`. Round-trip

- **2F done (ADR)**: `docs/decisions/006-federation-lww.md` captures
  the LWW rule for KV materialized views under federation (time,
  UUIDv7 lexicographic tiebreak). Implementation in the KV view is
  already deterministic under this rule today — new events with a
  greater `(time, id)` overwrite the KV row, and UUIDv7 ordering is
  lexicographic.

### Phase 1A — landed

- `ctxd-store-core` crate owns the `Store` trait, shared DTOs, and the conformance
  testsuite under `crates/ctxd-store-core/src/{lib.rs,testsuite.rs}`.
- `ctxd-store-sqlite` implements `Store` for `EventStore` via
  `crates/ctxd-store-sqlite/src/store_trait.rs`. The concrete `EventStore` API
  is unchanged — downstream crates (`ctxd-mcp`, `ctxd-http`, `ctxd-cli`,
  `ctxd-adapter-fs`) still import the concrete type from the `ctxd-store`
  re-export shim. Migrating those to `Arc<dyn Store>` is deferred; federation
  work in Phase 2 can depend on the trait directly where it helps.
- `crates/ctxd-store/` is a minimal back-compat shim that re-exports from
  `ctxd-store-sqlite`. No runtime code changed for downstream crates.
- Conformance test at `crates/ctxd-store-sqlite/tests/conformance.rs` runs
  every shared test against an in-memory SQLite store. Add new trait methods
  by (a) adding to `Store` in `ctxd-store-core/src/lib.rs`, (b) adding a
  conformance test in `ctxd-store-core/src/testsuite.rs`, and
  (c) wiring the impl in `ctxd-store-sqlite/src/store_trait.rs`.
