- **4D done (GitHub adapter)**: `crates/ctxd-adapter-github` now
  ships a real PAT-authed poller. Modules: `config`, `state`
  (sqlx-sqlite cursors + ETags + seen-resources), `client`

- **4A real embedders (DONE — Phase 4 v0.3)**: `OpenAiEmbedder`

  `/api/embeddings` endpoint). CLI: `--embedder {null|openai|ollama}`,
  `--embedder-model`, `--embedder-url`, `--embedder-api-key`. The
  registry that maps choice → `Arc<dyn Embedder>` lives at
  `ctxd_cli::embedder` and is exercised by
  `crates/ctxd-cli/tests/embedder_registry.rs`.
  Wiremock-backed integration tests cover the happy path, 429 +
  Retry-After backoff, batch chunking, 4xx error surface, and
  empty-input rejection.
- **4B HNSW persistence + hybrid search (DONE — Phase 4 v0.3)**:
  Vector view migrated from `instant-distance` to `hnsw_rs` 0.3.
  Files: `<db>.hnsw.{graph,data,meta,map}`. Magic precheck
  catches corruption before `hnsw_rs::HnswIo::init` panics. Open +
  rebuild path triggers when sidecars are missing, corrupt, or
  the SQL row count disagrees with the on-disk element count.
  RwLock guards Inner state so reads are concurrent and writes
  exclusive. Auto-flush every 1000 inserts (configurable). Tests:
  `vector_persist_and_restart`, `vector_corrupt_index_rebuild`,
  `vector_crash_recovery`. `ctx_search` accepts
  `search_mode: fts|vector|hybrid`; default is `hybrid` when an
  embedder is configured. Hybrid uses Reciprocal Rank Fusion
  (`k = 60`, the original 2009 paper value); see ADR 015.
  Tests: `search_mode_fts`, `search_mode_vector`,
  `search_mode_hybrid_rrf`. Bench at N=10k: HNSW 601 µs vs brute
  force 49 ms vs FTS 2.1 ms vs hybrid 3.3 ms.

### Phase 3 — 3A/3B/3C landed

- **3A done (trait + impls)**: `ctxd_cap::state::CaveatState` trait with
  `budget_increment`, `budget_get`, `rate_check`, `approval_request`,
  `approval_status`, `approval_decide`, **plus the new
  `approval_wait`** (default impl polls; in-memory overrides with
  `tokio::sync::Notify`). Two impls:
  - `ctxd_cap::state::InMemoryCaveatState` — Notify-backed approval
    wait so multiple concurrent waiters all wake on a single decide.
  - `ctxd_store_sqlite::caveat_state::SqliteCaveatState` — persists to
    `token_budgets` + `pending_approvals`. `approval_decide` is now
    a conditional `UPDATE … WHERE decision='pending'` so two racers
    only see one win, and the second `approval_decide` returns
    `CapError::Denied` (no flipping Deny → Allow after the fact).
- **3B done (BudgetLimit enforcement)**: `OperationCost(u64)` lives
  in `ctxd_cap::state` with documented per-op constants
  (read=0, write=1_000, search=1_000, entities=500, related=500,
  timeline=2_000). `CapEngine::mint_full` accepts
  `Option<&BudgetLimit>` and emits the `budget_limit/2` fact.
  `CapEngine::verify_with_state` charges the per-op cost atomically
  via `CaveatState::budget_increment` and returns
  `CapError::BudgetExceeded { currency, spent, limit }` on overshoot.
  Reserve-then-commit semantics — see ADR 011 for the trade-off.
- **3C done (HumanApprovalRequired)**: `requires_approval(op)` fact
  emitted at mint time. Verifier creates a UUIDv7 approval, calls
  `approval_request`, awaits `approval_wait(timeout)`, returns
  `CapError::ApprovalDenied` / `CapError::ApprovalTimeout` on the bad
  paths. CLI: `ctxd approve --id <uuid> --decision allow|deny`.
  HTTP: `POST /v1/approvals/:id/decide` + `GET /v1/approvals`. The
  daemon constructs `Arc<dyn CaveatState> = SqliteCaveatState::new(store)`
  once at startup and shares it with HTTP and MCP. A
  `tokio::sync::broadcast::<PendingApproval>` channel is wired so a
  future notifier adapter can subscribe — no adapter ships in v0.3.
- **Tracing**: approval request/decide emit structured `tracing::info!`
  with `token_id`, `approval_id`, `operation`, `subject`, `decision`.
  Tokens themselves are never logged.
- **ADRs**: 011 (caveat-state wiring + cost table) and 012 (approval
  flow). Both landed in this branch.
- **Tests added** in this branch (all green):
  1. `crates/ctxd-cap/tests/budget_limit_enforcement.rs`
  2. `crates/ctxd-store-sqlite/tests/budget_limit_persists_across_restart.rs`
  3. `crates/ctxd-cap/tests/budget_not_enforced_when_state_none.rs`
  4. `crates/ctxd-cap/tests/approval_request_then_allow.rs`
  5. `crates/ctxd-cap/tests/approval_request_then_deny.rs`
  6. `crates/ctxd-cap/tests/approval_timeout.rs`
  7. `crates/ctxd-http/tests/approval_http_endpoint.rs`
  8. `crates/ctxd-cli/tests/approval_cli_subcommand.rs`
  9. `crates/ctxd-cap/tests/budget_cost_table.rs`
- **Still deferred** (owned by other agents per the multi-worktree
  plan):
  - **3D multi-transport MCP**: SSE + streamable-HTTP. Owner: a
    different worktree.
  - **3E rate-limit state**: `CaveatState::rate_check` is still a

  `PeerCursorRequest`, `PeerCursor`, `PeerFetchEvents`. Round-trip
